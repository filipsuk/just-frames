# Agent Instructions

## General
- Keep changes minimal and focused.
- Prefer small, testable functions and clear naming.
- Use TypeScript for source code.
- Follow TDD when implementing new logic (write tests first).

## Commits
- Always use Conventional Commits (e.g., `docs: ...`, `feat: ...`, `chore: ...`).

## Testing
- When applicable, run relevant tests and report the exact commands.

## Screenshots
- When making changes to the preview screen layout/styles, capture a Playwright screenshot like the most recent preview layout example.
- Capture screenshots using an iPhone 12 resolution (390x844).
